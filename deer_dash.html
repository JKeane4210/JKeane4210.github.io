Deer Dash

During the summer of 2019, my friend Kacper Mocarski wanted to work on a new project and learn something new while the world was in lockdown in the beginning months of the pandemic. We decided to make a project in the game engine Unity, meaning we would have to learn how to code in C#. We began with several zoom calls trying to come up with ideas. We applied the design thinking mindset by asking other students in our school what makes games addicting, as our target audience was teenagers. Eventually, we decided on making a 3D game where the player would be driving down a highway where there are a lot of deer (or other animals) crossing the road. You would have to manage your fuel and your car's health while picking up power ups to see how long you could last.

Through this, we also learned how to design our own 3D models for cars and animals in the 3D modeling software blender. I ended up making several animals, landscapes, and cars that ended up going into the final game. This experience was valuable because it showed the amount of work that goes into making a videogame because in between sprites, graphics, music, sound effects, and functionality, there is so much that goes into making a great game. We did not end up finishing all the functionality that we wanted to have a polished product, so we currently just have exported versions of the app on our phones.

Project Analysis:

What did you do well in the project?

In this project, I did well at seeking out good sources to learn from because there was so many new things that had to be learned for this. I had to learn C# for programming, Unity for using a game engine, and Blender for creating 3D models for the game. I was able to "learn how to learn" really well through this project. Additionally, I did well at coordingating working with my friend to make sure that we each had features we could keep working on.

What new skills/knowledge did you gain from working on the project? What old skills/knowledge did you revisit or improve?

I learned how to use the game engine Unity from this, which is a very powerful engine, which can be used to created industry standard games. Inherently from this, I learned how to program in C#, which was new language for me. I realized that C# has a lot of similarities to Java, which I knew at the time, so I was able to apply my understanding of Java to speed up my learning. Finally, I learned how to create 3D models in Blender, which was completely new for me.

What went well and what opportunities did you identify for improvement in your next project? Look for opportunities to identify links between projects.

I think the best part of this project is that Kacper and I learned so many new things over the course of the summer, and it was even better that we were able to teach ourselves. I liked trying to manage every aspect of the game because it made me realize the complexity of creating a full-blown game.

However, because we did not publish this project by the end of the summer, we have a 90% finished project that is sitting on the shelf because of how busy we are with school now. Unlike my LinkedIn project, there is no real driving force that is keeping us focused on continuing to develop. In the future, I feel like having a stricter release date could have gotten us to a finished product.
