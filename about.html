Design Rationale

Contrast

  • Font Sizes

    I chose to have strongly contrasting font sizes to define the hierarchy of my project pages. I had larger fonts to highlight the start of sections in the content. For example, with the analysis portions of my project pages, I would have the biggest fonts for the main components, a medium-size font for the subsections, and my smallest font size for the actual content.

  • Project Blocks

    On the "Projects" tab, I chose to have distinct blocks to highlight how many projects I have and what content is associated with which title.

  • Subheader Bar

    I chose to have a subheader bar pop up when the user enters the "Projects" or "Work Experience" page. This bar has a distinct color to highlight the bar holds different content from the header bar.

Repetition

  • Blocks

    On both my "Projects" page and my "Work Experience" page, I created the same layout for all of my projects/internships. Therefore, after using one block, the navigation becomes intuitive for the rest.

  • Project Pages Layout

    Each of my project pages has the same content layout, so users learn to read project pages efficiently after visiting one of the project pages.

Alignment

  • Project Page/Design Rationale Alignment

    To make the hierarchy of content clearer, I made a conscious effort to make the deeper subsections left-aligned with more indentation than their parent section.

Proximity

  • Programming Languages vs. Tools

    On my home page, I wanted to differentiate between the programming languages and industry tools I knew. To do this, I created two distinct clusters of my "skill cards" to show these groupings.

  • Project/Internships Blocks

    I put the description and time frame close to the title and icon for each of my project/internship blocks to make the content groupings clear to the user.

Catering to Users

  • Resume & Contact Information

    The user I wanted to focus on providing information for was employers looking to hire me for an internship/job. I chose to put "Resume" & "Contact Me" links in my main header bar to make finding these important pieces of information as easy as possible for employers.

  • Skills

    As a computer science student, employers need to know that I have foundations in certain languages/tools to be a viable candidate for jobs/internships. I chose to make these skills as obvious as possible by having my "skills cards" on the home page of my site.
